# Repository Working Rules

- Do not create, modify, or delete files outside this repository.
- Do not modify system-wide or global development environments.
- Never commit secrets, credentials, session values, or private keys.
- Protect every state-changing request with CSRF validation.
- Enforce authentication and object-level authorization on the server.
- Validate all user-controlled input on the server.
- Do not construct dynamic SQL by concatenating strings.
- Do not misuse HTML `safe` or `Markup` to bypass escaping.
- Do not delete, skip, or weaken tests to obtain a passing result.
- Update requirements, tests, and security documentation together with feature changes.
- Commit or push only when the user explicitly requests it.
- Never describe a failed or uncertain result as successful.

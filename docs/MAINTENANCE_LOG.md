# Maintenance Log

운영·테스트에서 발견한 결함이 어느 SDLC 단계로 피드백되었는지 기록하는 템플릿이다.

| 발견 일시 | 문제 | 사용자 영향 | 원인 | 돌아간 개발 단계 | 변경 | 회귀 테스트 | 결과 |
|---|---|---|---|---|---|---|---|
| YYYY-MM-DD HH:MM TZ | 식별자와 현상 | 영향 범위 | 확인된 근본 원인 | 요구사항/설계/구현/테스트 | 변경 요약과 참조 | 테스트 ID/명령 | Open/Verified |

현재 Phase 01에는 유지보수 사건을 등록하지 않았다. 실제 결과를 확인하기 전 `Verified`로
기록하지 않는다.

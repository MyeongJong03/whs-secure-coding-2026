# Requirements

## 범위와 용어

Phase 01은 안전한 실행 기반, 데이터 모델, 최초 migration, 문서와 foundation 테스트만
확정한다. 아래 업무 기능 요구사항은 공식 과제의 최소 범위를 추적하기 위한 것이며, 모델이
존재한다는 사실은 해당 서비스나 UI가 구현되었다는 뜻이 아니다. "가상 포인트"는 실제 금융
자산이나 결제 수단이 아닌 과제용 정수 값이다.

## 입력 정책

| 입력 | 정책 |
|---|---|
| 사용자명 | ASCII 영문 대·소문자, 숫자, 밑줄만 허용하며 4~32자 |

| 상품 설명 | 1~2000자 |
| 가격·송금액 | 1 이상의 정수 |
| 이미지 | JPEG, PNG, WebP만 허용 예정; SVG, GIF, 실행 가능 파일 금지 |
| 이미지 요청 | 전체 요청 최대 5 MiB, pixel 수와 처리 자원 제한 예정 |
| 채팅 메시지 | 1~500자 |
| 신고 사유 | 10~500자 |
| 검색 | 길이·페이지 크기 상한, 상태/정렬 allowlist, 정수 가격 범위 |

## Phase 01 제외 범위

Phase 01에는 회원가입·로그인·로그아웃 UI, 사용자 조회·프로필 변경, 상품 CRUD·검색·파일
업로드, 채팅 이벤트, 신고 처리, 송금 서비스, 관리자 UI가 포함되지 않는다. 이 문서의 예정
요구사항은 후속 구현 범위를 정의할 뿐 현재 전체 플랫폼 구현 완료를 의미하지 않는다.
